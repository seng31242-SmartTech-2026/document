# UC-05: Search and Filter Products

## Use Case ID
UC-05

## Use Case Name
Search and Filter Products

## Primary Actors
Buyer

## Preconditions
- Approved products exist.

## Postconditions – Success
- Matching products are displayed.

## Postconditions – Failure
- System shows no results message.

## Main Flow
1. Buyer enters keyword.
2. System searches catalogue.
3. System displays results.
4. Buyer applies filters.
5. System updates results.
6. Buyer sorts results.
7. Buyer selects product.

## Alternative Flows
### AF-01
Empty search displays all listings.

### AF-02
No matching products found.

## Business Rules
- Only approved products are shown.
- Search is case-insensitive.
