# UC-02: Login / Logout

## Use Case ID
UC-02

## Use Case Name
Login / Logout

## Primary Actors
Buyer, Seller, Admin

## Preconditions
- User account exists.
- User account is verified.

## Postconditions – Success
- User is authenticated.
- JWT token/session is created.

## Postconditions – Failure
- Login denied with error message.

## Main Flow
1. User opens login page.
2. User enters email and password.
3. System validates credentials.
4. System authenticates user.
5. System redirects user to dashboard.
6. User clicks logout.
7. System invalidates session/token.
8. User is redirected to homepage.

## Alternative Flows
### AF-01
Invalid credentials show error message.

### AF-02
Suspended account cannot log in.

## Business Rules
- JWT authentication must be used.
- Only verified accounts can log in.
