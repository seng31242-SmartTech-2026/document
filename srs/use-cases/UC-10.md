# UC-10: Track Order

## Use Case ID
UC-10

## Use Case Name
Track Order

## Primary Actors
Buyer, Seller

## Preconditions
- Confirmed order exists.

## Postconditions – Success
- Order status displayed.

## Postconditions – Failure
- Unauthorized access blocked.

## Main Flow
1. Buyer opens My Orders.
2. System displays orders.
3. Buyer selects order.
4. System shows tracking timeline.
5. Seller updates order status.
6. System sends notifications.
7. Buyer confirms delivery.

## Alternative Flows
### AF-01
Seller delay triggers reminder.

## Business Rules
- Order status moves only forward.
