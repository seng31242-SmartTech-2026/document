# UC-04: List a Product

## Use Case ID
UC-04

## Use Case Name
List a Product

## Primary Actors
Seller

## Preconditions
- Seller is logged in.
- Seller account is active.
- Seller completed KYC verification.

## Postconditions – Success
- Product listing created as Pending Review.

## Postconditions – Failure
- Product listing not created.

## Main Flow
1. Seller opens dashboard.
2. Seller clicks **Add New Listing**.
3. System displays listing form.
4. Seller enters product details.
5. Seller uploads images.
6. System validates information.
7. Seller submits listing.
8. System stores listing.
9. Admin receives notification.

## Alternative Flows
### AF-01
Seller saves listing as draft.

### AF-02
Invalid image format is rejected.

### AF-03
Admin rejects listing with reason.

## Business Rules
- Minimum 1 image required.
- Maximum 8 images allowed.
- Price must be greater than zero.
