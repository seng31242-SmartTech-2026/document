# UC-03: Reset Password

## Use Case ID
UC-03

## Use Case Name
Reset Password

## Primary Actors
Buyer, Seller, Admin

## Preconditions
- User has a registered email.

## Postconditions – Success
- Password is updated successfully.

## Postconditions – Failure
- Password remains unchanged.

## Main Flow
1. User clicks **Forgot Password**.
2. System asks for email.
3. User enters registered email.
4. System sends reset link.
5. User clicks reset link.
6. User enters new password.
7. System validates password.
8. Password is updated.

## Alternative Flows
### AF-01
Expired reset token displays error.

### AF-02
Invalid email displays notification.

## Business Rules
- Reset token expires after 30 minutes.
- Token is single-use only.
