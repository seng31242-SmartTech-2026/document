# UC-08: Checkout and Place Order

## Use Case ID
UC-08

## Use Case Name
Checkout and Place Order

## Primary Actors
Buyer

## Preconditions
- Buyer is authenticated.
- Cart contains items.

## Postconditions – Success
- Order created with Pending Payment status.

## Postconditions – Failure
- Order is not created.

## Main Flow
1. Buyer opens cart.
2. Buyer clicks checkout.
3. System displays order summary.
4. Buyer enters delivery details.
5. Buyer confirms order.
6. System validates stock.
7. System creates order.
8. Buyer redirected to payment gateway.

## Alternative Flows
### AF-01
Out-of-stock item prevents checkout.

### AF-02
Payment cancellation keeps order pending temporarily.

## Business Rules
- Stock reserved during payment.
