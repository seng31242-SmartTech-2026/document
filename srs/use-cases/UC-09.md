# UC-09: Process Payment

## Use Case ID
UC-09

## Use Case Name
Process Payment

## Primary Actors
Buyer, Payment Gateway

## Preconditions
- Pending payment order exists.

## Postconditions – Success
- Payment confirmed.

## Postconditions – Failure
- Payment failed.

## Main Flow
1. Buyer selects payment method.
2. Buyer enters payment details.
3. Payment gateway processes transaction.
4. Gateway sends webhook.
5. System verifies webhook.
6. Order status updated.
7. Buyer redirected to confirmation page.

## Alternative Flows
### AF-01
Transaction declined by bank.

## Business Rules
- Payment must be verified server-side.
