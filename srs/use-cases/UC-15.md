# UC-15: Admin – Manage Users

## Use Case ID
UC-15

## Use Case Name
Admin – Manage Users

## Primary Actors
Administrator

## Preconditions
- Admin is authenticated.
- At least one registered user exists in the system.

## Postconditions – Success
- User account is suspended or reinstated successfully.
- System updates user status.
- User receives notification about account status change.

## Postconditions – Failure
- Requested action is not completed.
- Admin receives an error message.

## Main Flow
1. Admin navigates to the Admin Panel.
2. Admin opens the User Management section.
3. System displays all registered users.
4. Admin filters users by type or status.
5. Admin selects a user account.
6. System displays user details and account status.
7. Admin chooses to Suspend or Reinstate the account.
8. System updates the user status.
9. System sends a notification to the affected user.
10. System displays confirmation message to Admin.

## Alternative Flows

### AF-01
If the selected user account does not exist, the system displays an error message.

### AF-02
If the Admin cancels the operation before confirmation, no changes are made.

### AF-03
If notification delivery fails, the account status is still updated and the failure is logged.

## Business Rules
- Only Administrators can manage user accounts.
- Suspended users cannot log in or perform transactions.
- All account status changes must be logged for audit purposes.
- Administrators cannot suspend their own accounts.
