# UC-14: Admin – Approve or Remove Listing

## Use Case ID
UC-14

## Use Case Name
Admin – Approve or Remove Listing

## Primary Actors
Administrator

## Preconditions
- Admin is authenticated.

## Postconditions – Success
- Listing approved, rejected, or removed.

## Postconditions – Failure
- Action not completed.

## Main Flow
1. Admin opens moderation queue.
2. System displays pending listings.
3. Admin reviews listing.
4. Admin approves or rejects listing.
5. System updates status.
6. Seller receives notification.

## Alternative Flows
### AF-01
Admin removes active listing.

## Business Rules
- Only admins can moderate listings.
- Rejection reason is mandatory.
